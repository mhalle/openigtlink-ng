## Session policy

**While only the upstream pin is available:**

- Correctness questions during spec development are answered by reading
  the upstream source at the pinned SHA.
- If upstream's behavior appears incorrect (crash on malformed input,
  silent acceptance of spec-violating data, etc.), the finding is
  flagged inline and recorded in [`../protocol/v3.md`](../protocol/v3.md)
  under a "Known upstream correctness issues" section (to be created
  when the first finding lands).
- Upstream bugs are **not silently worked around** in the spec. The
  spec describes what the library does today, with explicit notes
  where that differs from what it should do.
- Positive corpus generation is **deferred** until the patched oracle
  is pinned.

**Once both pins are available:**

- Positive-corpus bytes are generated by the patched oracle.
- Every positive entry is cross-checked against the upstream reference;
  any divergence is triaged as either (a) upstream bug (patched wins,
  documented in schema `legacy_notes`), (b) compat-load-bearing quirk
  (patched library gets a commit preserving it, corpus documents both
  behaviors), or (c) a hardening bug (patched library needs a fix).
- Negative-corpus entries record the patched library's rejection
  behavior as the contract, with the upstream library's behavior
  recorded for historical context.

**Long-term:**

- The hardening commits from the patched library land upstream.
- Post-merge, `openigtlink-upstream/` becomes the single oracle for
  both positive and negative corpus generation.
- The patched-vs-upstream dual-library harness in `corpus-tools/`
  collapses to a single-library harness.

## Re-cloning the upstream reference

```bash
cd corpus-tools/reference-libs
git clone https://github.com/openigtlink/OpenIGTLink.git openigtlink-upstream
cd openigtlink-upstream
git checkout 94244fed7051e00cbcd4c341a7d7656be8ac404a
```

Any clone at the pinned SHA is equivalent. If the clone is at a
different SHA, update this file or reset to the pin before using it as
a correctness reference.
